# カスタム指示

カスタム指示（Custom Instructions）は、ChatGPTに対してあなたの好みや要求を永続的に記憶させる機能です。毎回同じ指示を繰り返す必要がなく、一貫した品質の出力を得られます。

!!! tip "Plus以上の機能"
    カスタム指示はChatGPT Plus、Pro、Team、Enterpriseプランで利用可能です。無料プランでは設定できません。

## カスタム指示とは

### 基本概念

カスタム指示は、ChatGPTがあなたとの対話で常に参考にする「設定ファイル」のようなものです。以下の2つのセクションで構成されます：

1. **What would you like ChatGPT to know about you?**  
   あなた自身について（背景、役割、専門分野等）

2. **How would you like ChatGPT to respond?**  
   ChatGPTの回答スタイル・形式について

### システムプロンプトとの違い

| 項目 | カスタム指示 | システムプロンプト |
|------|--------------|-------------------|
| 設定方法 | GUI設定 | 各会話で指定 |
| 持続性 | 永続的 | 会話ごと |
| 変更頻度 | 少ない | 頻繁 |
| 用途 | 基本的な振る舞い | 特定タスク用 |

## 設定方法

### アクセス方法
1. ChatGPT画面右下のプロフィール（名前部分）をクリック
2. 「Settings」→「Personalization」→「Custom instructions」
3. 有効化して2つのテキストボックスに記入

### 文字数制限
- 各セクション：1,500文字まで
- 合計：3,000文字以内

!!! warning "制限について"
    文字数制限があるため、最も重要な情報を厳選して記載しましょう。長すぎると効果が薄れる可能性があります。

## 効果的な書き方のコツ

### 1. 具体性を重視

**❌ 曖昧な例**
```
仕事で使うので、ビジネス的な回答をしてください
```

**✅ 具体的な例**
```
私はWebマーケティング部門のマネージャーです。
主な業務：SEO、広告運用、アクセス解析、コンテンツ戦略
扱う予算：月500万円規模
チーム規模：5名
重視する指標：ROI、CPA、オーガニック流入数
```

### 2. 階層化して整理

```markdown
## 基本情報
- 職種：[具体的な役職・職種]
- 業界：[業界・会社規模]
- 専門分野：[得意領域・資格等]

## 業務内容
- 主要業務：[日常的な作業内容]
- 意思決定レベル：[権限範囲]
- 関係者：[主要なステークホルダー]

## 価値観・優先事項
- 重視する要素：[品質・速度・コスト等]
- 避けたい要素：[リスク・制約等]
```

### 3. 出力形式の詳細指定

```markdown
## 回答スタイル
- 文体：です・ます調、ビジネス文書風
- 構成：結論→理由→具体例→次のアクション
- 情報量：概要→詳細の2段階構成
- 視覚化：箇条書き・表を積極活用

## 必須含有要素
- 実行可能なアクションプラン
- リスクや注意点の明記
- 関連リソースや参考情報
- ROI・効果測定の観点
```

## 用途別テンプレート

### 開発者向け

```markdown
【What would you like ChatGPT to know about you?】
私はフルスタック開発者です。

## 技術スタック
- フロントエンド：React, TypeScript, Next.js
- バックエンド：Node.js, Python (Django/FastAPI)
- データベース：PostgreSQL, MongoDB
- インフラ：AWS (EC2, S3, RDS), Docker
- その他：Git, GitHub Actions, Vercel

## 経験・専門性
- Web開発歴：5年
- 得意分野：SPA開発、REST API設計、パフォーマンス最適化
- 現在の役割：小規模チーム（3名）のテックリード
- 主要業務：機能開発、コードレビュー、技術選定

## 開発環境・制約
- 主要エディタ：VS Code
- 開発OS：macOS
- プロジェクト規模：中小規模SaaS
- セキュリティ要件：中程度（個人情報取扱いあり）

【How would you like ChatGPT to respond?】
## 回答スタイル
- 技術的精度を最重視
- 複数のアプローチを提示（pros/cons含む）
- セキュリティ・パフォーマンスの観点を必ず含める
- 実装例は実際に動作するコードを提供

## コード出力要件
- TypeScript優先（可能な限り）
- エラーハンドリングを含める
- コメントは重要な部分のみ（過度に冗長にしない）
- テストコード例も可能な限り提供

## 説明の構成
1. 概要・結論
2. 実装方法（ステップバイステップ）
3. コード例
4. 注意点・ベストプラクティス
5. 参考リンク（可能であれば）

## 避けてほしいこと
- 理論のみで実装方法がない回答
- 古い技術・非推奨の手法
- セキュリティリスクのあるコード例
```

```markdown
私はWebメディアのコンテンツライター・編集者です。

## 専門分野・経験

- 専門：ビジネス、マーケティング、テクノロジー分野
- 媒体：企業オウンドメディア、業界専門誌、Webメディア
- 月間執筆：約20記事（2,000-5,000字/記事）

## 対象読者
- 主要ターゲット：BtoB企業の決裁者・担当者
- 業界：IT、製造業、サービス業
- 職種：経営者、マネージャー、専門職
- 課題：業務効率化、コスト削減、売上向上

## 重視する要素
- 実用性：すぐ実践できる具体性
- 信頼性：データ・事例に基づく内容
- 読みやすさ：忙しい読者でも理解しやすい
- SEO：検索意図に合致した構成

## 制約条件
- コンプライアンス：薬機法、景表法に注意
- クライアント要望：業界慣例の尊重
- 納期：タイトなスケジュールが多い

【How would you like ChatGPT to respond?】
## 文章スタイル
- 文体：です・ます調（親しみやすさと信頼感のバランス）
- 構成：結論先行型、PREP法基準
- 情報密度：具体例・数値を豊富に含める
- 読みやすさ：適度な改行、見出し活用

## 記事構成要件
1. キャッチーな導入（問題提起・共感）
2. 記事の概要・読むメリット明示
3. 本論（3-5つのポイント、それぞれに具体例）
4. 実践的なアクションプラン
5. まとめ（要点整理・行動喚起）

## 必須要素
- 具体的な数値・統計データ
- 企業事例（成功・失敗両方）
- チェックリスト・フレームワーク
- 関連資料・リンクの提案
- SEOキーワード（自然な配置）

## 品質基準
- ファクトチェック：情報の正確性重視
- 独自性：ありきたりでない切り口
- 実用性：読後すぐ実行できるレベル
- エビデンス：主張には根拠を明記
```

### 学習者向け

```markdown
【What would you like ChatGPT to know about you?】
私はプログラミングを学習中の初心者です。

## 学習状況
- 学習期間：6ヶ月
- 現在の言語：JavaScript（基礎レベル）
- 学習目的：Webアプリケーション開発
- 最終目標：フリーランスエンジニアとして独立

## 現在のスキルレベル
- HTML/CSS：中級（レスポンシブ対応可能）
- JavaScript：初級（DOM操作、基本的なイベント処理）
- React：学習開始段階
- バックエンド：未学習
- Git：基本的な操作のみ

## 学習環境
- 主要学習方法：オンライン教材、実践プロジェクト
- 学習時間：平日2時間、休日5時間
- 開発環境：VS Code、Chrome DevTools
- 質問できる環境：限定的（主に独学）

## 課題・困りごと
- エラーの原因究明が苦手
- 「なぜそうなるか」の理解が不十分
- ベストプラクティスがわからない
- 学習の優先順位付けに悩む

【How would you like ChatGPT to respond?】
## 説明スタイル
- レベル：初心者向け、専門用語は必ず説明
- アプローチ：段階的、体感的理解を重視
- 例え：日常生活の事例を多用
- 励まし：学習者のモチベーション維持を意識

## 技術説明の構成
1. 「なぜ必要か」の説明（目的・メリット）
2. 基本概念の説明（図解・例え使用）
3. 簡単な例から複雑な例へ段階的解説
4. よくある間違い・注意点
5. 練習問題・応用課題の提案

## コード例の要件
- シンプルで理解しやすいもの優先
- 行ごとのコメント（何をしているか）
- エラーパターンも併せて説明
- 段階的な改良例（Version1→Version2など）

## 学習サポート
- 関連する基礎知識の補足
- 次に学ぶべき内容の提案
- 復習用のチェックポイント
- 実践プロジェクトのアイデア
- モチベーション維持のアドバイス

## 避けてほしいこと
- 高度すぎる内容・概念の突然の導入
- 「当たり前」として省略される説明
- 完成形のみ提示（プロセスが見えない）
- 専門用語の羅列
```

## 設定時の注意点

### 1. 情報の更新

カスタム指示の内容は定期的に見直しましょう：

- **役職変更時**: 新しい責任範囲を反映
- **スキル向上時**: 習得した技術・知識を追加
- **業務内容変更時**: 新しい業務の特徴を記載
- **優先事項変化時**: 重視する要素の変更

### 2. プライバシー考慮

機密情報は記載を避けましょう：

- 具体的な会社名・プロジェクト名
- 財務情報・売上数字
- 顧客情報・個人情報
- 社内機密・未公開情報

### 3. バランス調整

カスタム指示が強すぎると柔軟性を損なう可能性があります：

- **厳格すぎる制約**: 創造性を阻害
- **曖昧すぎる指示**: 効果が薄い
- **長すぎる設定**: 読み込み負荷増
- **短すぎる設定**: 不十分な指示

## 効果測定と改善

### 設定効果の確認方法

1. **回答品質の向上**: 求める形式での出力増加
2. **指示の省略**: 毎回説明する手間の削減
3. **一貫性の向上**: 複数会話での品質安定
4. **効率性の向上**: タスク完了時間の短縮

### 改善のサイクル

```markdown
1. 設定 → 2. 利用 → 3. 評価 → 4. 修正 → 1に戻る

## 評価項目
- 期待した出力形式になっているか？
- 専門性のレベルは適切か？
- 不要な情報が含まれていないか？
- 重要な観点が抜けていないか？

## 修正のタイミング
- 週1回：軽微な調整
- 月1回：内容の見直し
- 四半期：大幅な更新検討
```

## まとめ

カスタム指示は、ChatGPTとの対話を大幅に効率化する強力な機能です。効果的に活用するためのポイント：

### 設定のベストプラクティス
- **具体性**: 曖昧な表現を避け、詳細に記述
- **構造化**: 見出し・箇条書きで整理
- **優先順位**: 最重要事項を先頭に配置
- **実用性**: 日常業務に直結する内容を重視

### 継続的改善
- **定期見直し**: 業務・スキル変化に応じて更新
- **効果測定**: 出力品質の変化を定量的に評価
- **フィードバック**: 同僚・チームメンバーからの意見収集

適切に設定されたカスタム指示により、ChatGPTはあなた専用の高性能アシスタントとして機能します。投資する時間に見合う大きな生産性向上が期待できるため、ぜひ積極的に活用してください。
