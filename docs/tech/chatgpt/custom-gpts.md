# カスタムGPTs

カスタムGPT（GPTs）は、特定の用途に特化したChatGPTを作成・共有できる機能です。GPT Storeを通じて世界中のユーザーが作成したGPTsを利用でき、自分でも専用のGPTを構築できます。

!!! info "2025年の大型アップデート"
    2025年6月より、GPTs作成時にo3、o4-mini等の高度なモデルも選択可能になりました。これにより、より高性能なカスタムGPTの構築が可能です。

## カスタムGPTsとは

### 基本概念

カスタムGPTsは、以下の要素を組み合わせて作成します：

1. **Instructions（指示）**: GPTの振る舞いを定義
2. **Knowledge（知識）**: ファイルアップロードによる専門知識
3. **Actions（アクション）**: 外部APIとの連携
4. **Model Selection（モデル選択）**: 用途に応じたモデル選択

### 通常のChatGPTとの違い

| 項目 | 通常のChatGPT | カスタムGPT |
|------|---------------|-------------|
| 専門性 | 汎用的 | 特化型 |
| 知識ベース | 訓練データのみ | 追加ファイル可 |
| 外部連携 | 限定的 | API連携可能 |
| 共有 | 不可 | GPT Store公開可 |
| カスタマイズ | カスタム指示のみ | 総合的設定 |

## GPT Builder：作成方法ガイド

### 1. 基本設定

#### アクセス方法
1. ChatGPT画面左サイドバー「Explore GPTs」
2. 「Create a GPT」ボタンをクリック
3. Create（対話形式）またはConfigure（設定形式）を選択

#### 基本情報設定
```yaml
名前: [GPTの名前（具体的で分かりやすく）]
説明: [機能・用途の簡潔な説明]
アイコン: [DALL-E生成またはアップロード]
```

### 2. Instructions（指示）設計

効果的なInstructionsの構成：

```markdown
# [GPT名]

## Role & Purpose（役割・目的）
あなたは[専門分野]の専門家として、[対象ユーザー]の[解決したい課題]をサポートします。

## Core Capabilities（中心機能）
1. [機能1]: [詳細説明]
2. [機能2]: [詳細説明]
3. [機能3]: [詳細説明]

## Response Style（回答スタイル）
- 文体: [です・ます調 / である調 / カジュアル等]
- 構成: [結論先行 / 段階的説明 / 具体例重視等]
- 長さ: [簡潔 / 詳細 / ユーザー依存]

## Process & Workflow（処理フロー）
1. ユーザーの要求を確認・整理
2. [具体的なステップ]
3. [出力形式]
4. フォローアップ質問の提示

## Constraints & Guidelines（制約・ガイドライン）
- 避けるべき内容: [NG事項]
- 必須確認事項: [重要なチェックポイント]
- エラー時の対応: [問題発生時の処理]

## Examples（使用例）
[実際の対話例を2-3パターン]
```

### 3. Knowledge（知識ベース）活用

#### アップロード可能ファイル形式
- **文書**: PDF, DOCX, TXT, MD
- **データ**: CSV, JSON, XLSX
- **コード**: Python, JavaScript, その他
- **画像**: PNG, JPG（図表・チャート）

#### 効果的なKnowledge設計

```markdown
## ファイル構成例（マニュアル作成GPT）

### 1. style_guide.pdf
- 文章スタイル規定
- 用語統一ルール
- 図表作成ガイドライン

### 2. template_collection.docx
- 各種文書テンプレート
- 構成パターン集

### 3. company_info.txt
- 会社概要・サービス情報
- 専門用語定義
- ブランドガイドライン

### 4. examples.json
- 良い例・悪い例のサンプル
- カテゴリ別事例集
```

!!! tip "Knowledge活用のコツ"
    - **ファイルサイズ制限**: 1ファイル最大512MB
    - **合計制限**: 20ファイルまで
    - **更新頻度**: 情報が古くなったら定期更新
    - **構造化**: 検索しやすい形式で整理

### 4. Actions（API連携）設定

外部サービスとの連携により、GPTの機能を大幅に拡張できます。

#### 一般的な連携例

```yaml
# Google Sheets連携
目的: データの読み書き、レポート生成
API: Google Sheets API
用途: 在庫管理、売上分析、顧客管理

# Slack連携  
目的: チームコミュニケーション
API: Slack Web API
用途: 通知送信、メッセージ投稿、ファイル共有

# GitHub連携
目的: コード管理、Issue管理
API: GitHub REST API
用途: プルリクエスト作成、コードレビュー
```

#### Action設定手順

1. **スキーマ定義**: OpenAPI形式でAPI仕様を記述
2. **認証設定**: API Key、OAuth等の認証方法
3. **テスト実行**: 動作確認とデバッグ
4. **エラーハンドリング**: 例外処理の実装

```json
{
  "openapi": "3.1.0",
  "info": {
    "title": "Sales Data API",
    "description": "売上データ取得・更新API",
    "version": "v1.0.0"
  },
  "servers": [
    {
      "url": "https://api.example.com"
    }
  ],
  "paths": {
    "/sales/monthly": {
      "get": {
        "description": "月次売上データ取得",
        "parameters": [
          {
            "name": "year",
            "in": "query", 
            "description": "対象年",
            "required": true,
            "schema": {
              "type": "integer"
            }
          }
        ]
      }
    }
  }
}
```

### 5. モデル選択（2025年新機能）

用途に応じて最適なモデルを選択できます：

| モデル | 適用場面 | 特徴 |
|--------|----------|------|
| GPT-5.2 | 汎用・創作 | 最高品質、バランス型 |
| GPT-4o | マルチメディア | 画像・音声対応 |
| o3 | 高度な推論 | 複雑な分析・計算 |
| o4-mini | 軽量タスク | 高速・低コスト |

```markdown
## モデル選択の指針

### 文章作成系GPT → GPT-5.2
理由: 創作性と品質のバランス

### データ分析系GPT → o3
理由: 深い思考・推論が必要

### 日常業務支援GPT → o4-mini  
理由: 高速レスポンス・コスト効率

### マルチメディア処理GPT → GPT-4o
理由: 画像・音声入力対応
```

## 公開・共有方法

### 公開レベルの選択

1. **Only me（自分のみ）**: 個人利用専用
2. **Anyone with the link（リンク共有）**: URL知ってる人のみ
3. **Public（一般公開）**: GPT Storeで検索可能

### GPT Store公開のメリット

- **利用者数拡大**: 世界中のユーザーにリーチ
- **フィードバック収集**: 使用感・改善点の把握
- **ブランディング**: 専門性のアピール
- **収益化**: 将来的な収益化の可能性

## 収益化（GPT Store）

- **基本条件**: Plus以上のプランユーザーの利用が対象

- **支払い**: 月次での分配

## まとめ

カスタムGPTsは、特定用途に最適化されたAIアシスタントを作成できる革新的な機能です。成功のためのポイント：

### 技術的ポイント
- **明確な専門性**: 特定領域に深く特化
- **知識ベース**: 豊富で正確な情報の蓄積
- **API連携**: 実用性を高める外部サービス連携
- **適切なモデル選択**: 用途に応じた最適化

### ビジネス的ポイント
- **ユーザー価値**: 明確で測定可能な価値提供
- **差別化**: 他にない独自性の確保
- **継続改善**: 利用データに基づく反復改善
- **コミュニティ**: ユーザーとの関係構築

2025年のモデル選択機能により、これまで以上に高性能なカスタムGPTの構築が可能になりました。自分の専門性を活かしたGPTsを作成し、世界中のユーザーの課題解決に貢献してみてください。
