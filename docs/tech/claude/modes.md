# モード・機能活用

> Claude の豊富な機能を最大限に活用するための完全ガイド。

## モデル別特徴・使い分け

### 🚀 Opus 4.6（最上位モデル）

**特徴**:
- 最高レベルの推論能力
- 長時間の思考が必要な複雑タスクに特化
- 1M トークンコンテキスト対応（ベータ）

**最適な用途**:
```
✅ 複雑な問題解決・戦略立案
✅ 高品質な創作・ライティング
✅ 詳細な分析・レポート作成
✅ 大規模コードベースの理解・リファクタリング
✅ 学術的な議論・研究支援
```

**実例**:
```
経営戦略の策定支援
「弊社（SaaS、従業員50名）の今後3年間の成長戦略を、
市場分析・競合分析・財務予測を含めて包括的に提案してください」
```

### ⚡ Sonnet 4.5（バランス型）

**特徴**:
- 速度と品質のバランスが良い
- 日常的なタスクに最適
- コストパフォーマンス◎

**最適な用途**:
```
✅ 日常的なコーディング
✅ 文書作成・編集
✅ データ分析
✅ 翻訳・要約
✅ 学習支援
```

**実例**:
```
Python コード生成
「CSVファイルを読み込んで売上データを可視化するダッシュボードを
Streamlit で作成してください」
```

### 🏃 Haiku 4.5（高速軽量）

**特徴**:
- 高速レスポンス
- 軽いタスクに最適
- コスト効率◎

**最適な用途**:
```
✅ 簡単な質問応答
✅ 短文の生成・修正
✅ 基本的なコード修正
✅ リアルタイム支援
✅ 大量の単純処理
```

## Extended Thinking（深い推論モード）

**Extended Thinking** は Claude 4シリーズの革新的機能で、AI が問題を段階的に深く考察するモードです。

### 💭 Extended Thinking の特徴

- **思考プロセスの可視化**: AI の推論ステップを確認可能
- **より深い分析**: 表面的でない洞察を提供
- **段階的解決**: 複雑な問題を分解して処理

### 🚀 活用場面

**複雑な問題解決**:
```
「新規事業のビジネスモデルを検討中です。
市場機会、競合状況、収益構造、リスク要因を
多角的に分析して戦略を提案してください」
```

**深い技術分析**:
```
「マイクロサービスアーキテクチャへの移行を検討しています。
現在のモノリシック構造の課題、移行手順、技術選択、
リスクを総合的に分析してください」
```

### ⚙️ 利用方法

1. **Claude Pro/Max プラン** で利用可能
2. 複雑な質問をする際に自動で発動
3. 「深く考えて」「段階的に分析して」等のトリガーフレーズ有効

## Artifacts（コード実行・可視化）

**Artifacts** は Claude の最強機能の一つで、対話内でインタラクティブなコンテンツを生成・実行できます。

### ✨ Artifacts でできること

#### 🐍 Python コード実行
```python
import pandas as pd
import matplotlib.pyplot as plt

# データ分析・可視化の実行例
data = pd.read_csv('sales_data.csv')
plt.figure(figsize=(10,6))
plt.plot(data['month'], data['revenue'])
plt.title('月次売上推移')
plt.show()
```

#### 🌐 Web アプリ作成
```html
<!-- インタラクティブなHTML/CSS/JSアプリ -->
<!DOCTYPE html>
<html>
<head><title>タスク管理アプリ</title></head>
<body>
    <div id="app">
        <input type="text" id="taskInput" placeholder="新しいタスク">
        <button onclick="addTask()">追加</button>
        <ul id="taskList"></ul>
    </div>
</body>
</html>
```

#### 📊 データ可視化
```javascript
// Chart.js を使用したグラフ生成
const ctx = document.getElementById('chart').getContext('2d');
new Chart(ctx, {
    type: 'bar',
    data: chartData,
    options: { responsive: true }
});
```

### 🚀 Artifacts 活用実例

**データ分析ダッシュボード**:
```
「過去3年間の売上データを分析して、
月次推移、商品別売上、地域別比較を
インタラクティブに表示するダッシュボードを作ってください」
```

**プロトタイプ作成**:
```
「タスク管理アプリのプロトタイプを作成してください。
追加・削除・完了機能付きで、使いやすいUIにしてください」
```

## Projects（プロジェクト機能）

**Projects** は特定用途に特化した Claude 環境を作成する機能です。

### 🎯 Projects の価値

- **一貫性**: 同じ設定で継続作業
- **効率性**: 毎回の指示が不要
- **専門性**: 特化した能力を発揮

### 📚 Project の構成要素

#### 1. **Project Instructions（カスタム指示）**
```
あなたは経験豊富なPythonエンジニアです。
以下のスタイルでコードを作成してください：
- PEP8準拠
- 型ヒント必須
- docstring付き
- エラーハンドリング充実
- テストコード提供
```

#### 2. **Knowledge（ナレッジベース）**
- 関連ドキュメント
- コードベース
- 設計書
- API仕様書

[詳細: Projects・カスタム指示](projects.md)

## ファイルアップロード・分析

### 📄 対応ファイル形式

| 形式 | 用途 | 特徴 |
|------|------|------|
| **PDF** | 文書・論文・資料 | OCR対応、構造化解析 |
| **CSV/Excel** | データ分析 | 統計分析、可視化 |
| **画像** | Vision分析 | 内容認識、OCR、図表解析 |
| **コード** | プログラム解析 | レビュー、リファクタリング |

### 🚀 活用実例

**大容量文書の解析**:
```
「この300ページの技術仕様書を分析して、
要点を整理した10ページのサマリーを作成してください」
```

**データ分析**:
```
「アップロードした売上データから、
トレンド分析と来期予測を行ってください」
```

### 💡 200K〜1M コンテキスト活用法

**長文コンテキストの強み**:
- 大量の関連文書を一度に処理
- コードベース全体の理解
- 詳細な分析・比較

**実践的な使い方**:
```xml
<documents>
<document name="仕様書A">
[大量テキスト]
</document>
<document name="仕様書B">
[大量テキスト]
</document>
</documents>

上記の仕様書を比較分析して、
差分と移行時の注意点をまとめてください。
```

## Computer Use（AI画面操作）

**Computer Use** は AI がコンピューターの画面を見て実際に操作する革新的機能です。

### 🖱️ できること

- **画面認識**: スクリーンショットの理解
- **操作実行**: クリック、タイピング、スクロール
- **アプリ制御**: ブラウザ、ソフトウェアの操作
- **タスク自動化**: 反復作業の自動実行

### ⚠️ 現在の制限

- ベータ機能（限定アクセス）
- 実行環境に依存
- セキュリティ考慮が必要

### 🚀 活用可能性

```
「この Web フォームに顧客データを入力して、
PDFレポートをダウンロードしてください」

「スプレッドシートからデータを読み取って、
専用システムに入力してください」
```
